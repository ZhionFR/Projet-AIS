Tableau de claude des commandes principales utilisées et leur raison d'échec :

|Commande|Raison d'échec|
|---|---|
|`netplan`|Netplan n'est pas installé sur Parrot OS (utilise ifupdown classique)|
|`sudo touch /etc/netplan/00-installer-config.yaml`|Le répertoire `/etc/netplan/` n'existe pas sur Parrot|
|`sudo systemctl restart networking` (avec wpa-psk en clair)|`wpa_supplicant` a échoué à initialiser le driver : "Failed to initialize driver interface"|
|`sudo nmcli connection add ... wifi-bridge`|Type de connexion `wifi-bridge` n'existe pas dans NetworkManager (types valides : wifi, ethernet, bridge, bridge-slave, etc.)|
|`sudo nmcli connection add ... 802-11-wireless-security.key-mgmt`|Paramètre invalide : la clé correcte est `802-11-wireless-security` (wifi-sec)|
|`sudo nmcli connection up wifi-ssid`|NetworkManager tentait d'utiliser `enp0s31f6` (Ethernet) au lieu de `wlp3s0` (WiFi) → interface incompatible|
|`sudo nmcli device wifi connect ... ifname wlan0`|Interface incorrecte : la vraie interface est `wlp3s0`, pas `wlan0`|
|`sudo ip link set wlp3s0 master br0`|**Driver WiFi refuse le mode bridge natif** : "Device does not allow enslaving to a bridge"|
|`sudo nmcli connection add ... slave-type bridge`|Conflit : NetworkManager ne pouvait pas initialiser le WiFi tandis qu'il était en attente d'esclave du bridge|
|`sudo dhclient br0`|`dhclient` n'est pas installé sur Parrot (utilise `dhcpcd` à la place)|
